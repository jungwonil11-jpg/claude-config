# 노션정리 — 프로젝트 코드를 노션 학습 노트로 동기화

현재 프로젝트의 코드 파일을 분석해 노션에 학습 노트를 생성·수정·보완한다.

## 노션 페이지 구조 (React 강의 노트와 동일 패턴)

```
상위 페이지 (예: 🐍 Python 강의)
  └── day01
  └── day02
       └── 주제별 섹션 (문자열, 리스트 등)
```

## 작성 스타일
- **음슴체** 사용 (예: "~임", "~함", "~됨")
- **비유** 적극 활용 — 낯선 개념은 익숙한 것에 빗대어 설명
- **표** 사용 — 비교, 속성 정리에 적합할 때
- **Callout 블록** 사용 — 핵심 요점, 주의사항, 팁 강조에 활용
- 코드 예제는 코드 블록으로 포함

## 각 하위 페이지 내부 구조
1. 개념 설명 (H2)
2. 핵심 규칙/문법 (H2 → H3 번호 목록)
3. 실습 코드 분석 (H2, 코드 블록)
4. 핵심 포인트 정리 (H2, 표)
5. 주의사항/팁 (Callout)

## 실행 순서

### 1. 프로젝트 파악
현재 프로젝트의 수업일별 폴더(`day01/`, `day02/`, ...)와 예제 파일(`ex01_*.py` ...)을 읽어 어떤 개념이 다뤄졌는지 파악.

### 2. 상위 페이지 찾기
노션에서 프로젝트명으로 검색 (예: "Python 강의").
- 있으면 → 해당 페이지 사용
- 없으면 → 새로 생성

### 3. day별 하위 페이지 동기화
각 dayXX 폴더마다:
- 노션에 해당 페이지(예: `day01`) 있는지 확인
  - 없으면 → 생성
  - 있으면 → 기존 내용과 코드 파일 비교 후 누락/변경 항목만 보완

### 4. 내용 작성 기준
- 코드 파일에서 **개념** 추출 (주석, 함수명, 예제 패턴 기반)
- 강사 코드 원본의 흐름을 따라 설명
- 실습 코드는 핵심 예제만 1~2개로 압축

### 5. 완료 보고
- 생성된 페이지 목록
- 수정된 페이지 목록
- 변경 없이 스킵한 페이지 목록
- 각 페이지 노션 링크 제공
